# Programnesia
Aplikasi Belajar Pemograman
Portal Knowledge Base & Learning Center yang mengumpulkan materi pemrograman dari berbagai sumber terpercaya di internet, mengelompokkannya berdasarkan kategori, tingkat kesulitan, dan teknologi, serta menyediakan CMS yang lengkap untuk mengelola konten, iklan, statistik, dan monetisasi.
Saya juga melihat ada beberapa tantangan penting yang perlu diperhatikan sejak awal.
Hal yang perlu diperhatikan terkait scraping
Untuk berita terbaru, melakukan scraping umumnya masih memungkinkan jika mengikuti ketentuan situs yang menjadi sumber.
Namun untuk artikel pembelajaran, menampilkan hasil scraping secara utuh dari website lain dapat melanggar hak cipta dan syarat penggunaan situs tersebut. Karena itu saya menyarankan pendekatan yang lebih aman dan profesional:
•	Scraping hanya mengambil metadata: 
o	Judul 
o	Ringkasan 
o	Thumbnail 
o	Kategori 
o	Penulis (jika tersedia) 
o	Tanggal 
o	URL sumber 
•	Sistem dapat membuat indeks materi dan mengarahkan pengguna ke artikel asli, atau Anda menambahkan penjelasan/original content sendiri yang merujuk sumber tersebut. 
•	Untuk sumber yang memang menyediakan lisensi terbuka (misalnya dokumentasi resmi, repositori GitHub, tutorial open-source), kontennya dapat diintegrasikan sesuai lisensinya. 
Dengan pendekatan ini, website akan lebih aman secara hukum dan tetap bernilai tinggi.

Optimasi Performa
Karena jumlah artikel bisa mencapai ratusan ribu, arsitektur pencarian harus dioptimalkan:
•	Full-Text Search MySQL untuk judul, ringkasan, dan tag. 
•	Indeks pada kolom kategori, level, tanggal, status, dan popularitas. 
•	AJAX Search dengan debounce agar tidak mengirim request di setiap penekanan tombol. 
•	Server-side pagination (LIMIT dan OFFSET) untuk daftar artikel. 
•	Lazy loading gambar dan infinite scroll pada halaman tertentu. 
•	Cache kategori, tag, roadmap, dan menu navigasi. 
•	Optimasi gambar ke format WebP dan pembuatan thumbnail otomatis. 
•	Jika skala sudah sangat besar (jutaan artikel), siapkan lapisan pencarian menggunakan Elasticsearch atau OpenSearch agar pencarian tetap cepat.

26. Rekomendasi Arsitektur Database
Saya menyarankan beberapa penyempurnaan dibanding rancangan awal agar lebih siap untuk skala besar:
•	Gunakan InnoDB pada seluruh tabel untuk mendukung transaksi dan foreign key. 
•	Seluruh primary key menggunakan BIGINT UNSIGNED AUTO_INCREMENT, kecuali tabel master kecil (kategori, level, role) yang dapat menggunakan INT. 
•	Terapkan soft delete (deleted_at) pada tabel penting seperti articles, downloads, users, dan comments agar data tidak hilang saat dihapus. 
•	Tambahkan UUID sebagai public_id untuk URL publik atau API sehingga ID internal tidak terekspos. 
•	Pisahkan visitor logs, activity logs, dan search logs dari tabel transaksi utama agar tidak mengganggu performa. 
•	Simpan ringkasan statistik (jumlah komentar, rating rata-rata, jumlah bookmark, jumlah unduhan) langsung pada tabel articles dan downloads, lalu perbarui secara asynchronous menggunakan cron atau queue.

BAB 20. Rekomendasi Arsitektur Jangka Panjang
Agar BelajarTeknologi tetap mudah dikembangkan selama 5–10 tahun ke depan, saya menyarankan beberapa penyempurnaan di luar PRD dasar:
•	Gunakan Composer sebagai dependency manager meskipun memakai CI3. 
•	Seluruh konfigurasi lingkungan menggunakan file .env (melalui library yang kompatibel dengan CI3). 
•	Gunakan database migration dan database seeder sehingga struktur database dapat dikelola dengan versi kode. 
•	Pisahkan proses berat (scraping, generate sitemap, resize gambar, pengiriman email) ke queue worker, sehingga tidak menghambat request pengguna. 
•	Semua modul baru (misalnya AI Assistant, LMS, atau Mobile API) dibangun di atas Service Layer yang sama, sehingga dapat digunakan kembali tanpa menulis ulang logika bisnis.

Kesimpulan
Keempat volume PRD ini membentuk fondasi lengkap untuk membangun BelajarTeknologi sebagai portal pembelajaran teknologi berskala enterprise:
•	Volume 1: kebutuhan bisnis dan fungsional. 
•	Volume 2: desain database dan performa. 
•	Volume 3: desain UI/UX dan pengalaman pengguna. 
•	Volume 4: arsitektur teknis, standar implementasi, deployment, keamanan, dan roadmap. 
Dengan spesifikasi tersebut, proyek siap memasuki tahap berikutnya, yaitu Software Design Document (SDD) yang mendetail hingga level implementasi: struktur setiap modul, spesifikasi setiap controller, service, repository, API endpoint, sequence diagram, state diagram, dan pembagian pekerjaan pengembangan per sprint.
